# Лабораторна робота №1: Базові ідеї ООП

Цей невеликий проект є реалізацією покрокової рольової гри (RPG) для демонстрації основних принципів об'єктно-орієнтованого програмування: успадкування, поліморфізму, інкапсуляції та використання узагальнень (Generics).

## Виконання вимог

У проекті реалізовано всі ключові вимоги лабораторної роботи:

1.  **Класи (9 шт.):** `Entity`, `Unit`, `Warrior`, `Mage`, `Item`, `Weapon`, `Consumable`, `Inventory`, `BattleManager`.
2.  **Поля та методи:** Реалізовано понад 20 полів та 25 нетривіальних методів (логіка бою, лікування, інвентар, прокачка рівня).
3.  **Ієрархія успадкування:**
    *   `Entity` -> `Unit` -> `Warrior` / `Mage` (Глибина 3)
    *   `Item` -> `Weapon` / `Consumable` (Глибина 2)
4.  **Поліморфізм:**
    *   **Динамічний:** Метод `attack()` (різна реалізація для Воїна та Мага).
    *   **Динамічний:** Метод `use()` (різна поведінка для Зброї та Витратних матеріалів).
    *   **Статичний (Generics):** Клас `Inventory[T]`, що може зберігати як предмети, так і юнітів.

## Структура проекту

### Основні сутності (`Entity` -> `Unit`)
*   **Warrior:** Має високе здоров'я та захист. Використовує механіку "Люті" (`Rage`) для посилення атак. Має спеціальний удар `heavy_slam`.
*   **Mage:** Має менше здоров'я, але використовує "Ману" (`Mana`) для потужних заклять. Вміє медитувати для відновлення мани.

### Предмети (`Item`)
*   **Weapon:** Пасивні предмети, що мають міцність.
*   **Consumable:** Предмети одноразового використання (зілля здоров'я, зілля мани).

### Системи
*   **Inventory[T]:** Універсальне сховище з обмеженою місткістю.
*   **BattleManager:** Керує покроковим боєм між двома командами, розподіляє досвід (XP) та визначає переможця.

## Як запустити

Для запуску симуляції виконайте команду в терміналі:

```bash
python main.py
```

## Функціонал симуляції

При запуску `main.py` відбувається демонстрація:
1.  Створення персонажів та інвентарю.
2.  Додавання предметів у рюкзак.
3.  Використання предметів (лікування, відновлення мани).
4.  Демонстрація спеціальних атак.
5.  Автоматичний бій між командами.
6.  Виведення результатів та стану персонажів після бою.
